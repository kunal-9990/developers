Integrating Working Papers with Cloud

Integration with CaseWare Cloud provides additional functionality to Working Papers. Connecting these products enables the use of Cloud as a file management and storage center, while also providing new ways to track and complete your engagement online. When integrated in a terminal server environment, all staff that launch the application from the server are able to log in with their individual Cloud credentials.

To integrate Working Papers with Cloud:

  1. Ensure you have Working Papers 2016 or higher.

  2. Launch the Working Papers application.

  3. On the File menu, select Options | CaseWare Cloud.

  4. Ensure Integrate with CaseWare Cloud is selected. In the Server field, enter the URL of your organization’s CaseWare Cloud site. You do not need to enter the web protocol (http:// or https://). Click OK.

  5. A CaseWare Cloud Login prompt is displayed. Enter your User name and Password, then select OK.

Your copy of Working Papers is now integrated with Cloud.

Setting a default Cloud URL for your staff

Each staff member that launches the application from the terminal server will need to enter their login credentials and Cloud URL to connect to Cloud. Using a script, you can adjust the staff’s registry settings to automatically populate your Cloud URL details.

To set the default URL for your staff:

The registry key is located in:

  • HKEY_CURRENT_USER\SOFTWARE\CaseWare International\Working Papers\2016.00\Core

  1. Right-click the Server key and select Modify....

  2. In the Value data: field, enter your Cloud URL. Do not include the web protocol (http:// or https://) in the URL.

  3. Click OK to apply your changes.

Staff who open the Options | CaseWare Cloud menu will see the Cloud URL populated. They can enter their login credentials to complete integration.
