Deactivate or delete client entities

If you have client entities that are no longer active you can deactivate or delete them. Deactivating an entity preserves its data and prevents members of your organization from completing further engagement work. Deleting an entity removes it from Cloud entirely - along with all activities and files.

In most cases an entity should be deactivated rather than deleted.

Deactivate an entity

Deactivating an entity prevents your staff from accessing the entity while preserving all files and activities; entity data is difficult to recover after it is deleted. You can reactivate an inactive entity at any time, but if the entity is deleted it must be recreated.

Note: Entities with draft invoices cannot be made inactive. Any draft invoices should be finalized before making an entity inactive.

To deactivate an entity:

  1. Ensure you have the Entities Admin role or the equivalent privileges.

  2. Ensure that any interim invoices associated with the entity have been finalized. For more information on final invoices, see Create a final invoice.

  3. From the Cloud menu, select Entities.

  4. Select the entity you want to deactivate, then select Edit().

  5. From the sidebar, select Profile.

  6. From the Status drop-down menu, select Inactive.

  7. If the entity has associated billed or non-billable entries, complete one of the following:

    • Select Transfer unbilled…, then select Continue. For more information on transferring entries, see Transfer time or expense entries.

    • Select Delete unbilled…, then select Continue.

  8. Select Save.

The entity is deactivated. If you wish to reactivate the entity later, follow the steps below.

To reactivate an entity:

  1. Ensure you have the Entities Admin role or the equivalent privileges.

  2. From the Cloud menu, select Entities.

  3. Select the entity you want to deactivate, then select Edit().

  4. Select Profile.

  5. From the Status drop-down menu, select Active.

Delete an entity

Warning: In most cases, deleting an entity is not recommended. Instead, make the entity inactive. For more information, see Deactivate an entity.

In rare cases, you may want to completely remove an entity. For example, you may wish to delete an entity that was created by accident.

Note: Entities with associated invoices cannot be deleted, and should be deactivated instead.

To delete an entity:

  1. Ensure you have the Entities Admin role or the equivalent privileges.

  2. From the Cloud menu, select Entities.

  3. Select the entity you want to delete, then select More Actions () | Delete.

  4. If the entity has associated billed or non-billable entries, complete one of the following:

    • Select Transfer unbilled…, then select Continue. For more information on transferring entries, see Transfer time or expense entries.

    • Select Delete unbilled…, then select Continue.

  5. Select Yes.

You have deleted the selected entity. For information on creating and managing entities, see Create client workspaces.
