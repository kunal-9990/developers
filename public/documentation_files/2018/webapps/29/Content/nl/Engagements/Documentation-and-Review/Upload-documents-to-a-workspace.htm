You can add files and documents to Cloud so that all of the relevant material for your engagement is stored in one place, well organized, and easy to find and access.

Uploading a file

If you have any files that will be used as a reference during the engagement, you may want to upload them to Cloud. These files might include copies of correspondence with the client or files that the client sent for the engagement work. Your team members will be able to easily find the reference files and download local copies of them, if necessary.

To upload a file:

  1. Ensure that you have the Editor role or equivalent rights for the client entity.

  2. From the Cloud menu, select Files.

  3. Select New, and then choose Upload.

  4. In the Entity Selector dialog, select the entity that this file will belong to.

  5. In the File Upload dialog, select the Select Files button. Find the file in the Open dialog and click Open.

  6. In the Share Settings dialog, choose sharing options for the file, then select OK.

You have uploaded your file to the chosen client entity. If you limited the share settings for the file, you can share it with specific staff members right away, or you can wait until your team members need the file for their engagement work.

Uploading a file larger than 1GB

Files larger than 1GB must be uploaded with the Advanced Uploader. Using the Advanced Uploader requires Java and Microsoft Internet Explorer.

To upload a file larger than 1GB:

  1. Ensure that you have the Editor role or equivalent rights for the client entity.

  2. Ensure you have installed the latest version of Java, and are using a browser that supports it.

  3. From the Cloud menu, select Files.

  4. Select New, and then choose Upload.

  5. In the Entity Selector dialog, select the entity that this file will belong to.

  6. Select Use advanced uploader.

  7. Select the files you want upload, then select Add.

  8. In the Share Settings dialog, choose sharing options for the file, then select OK.

You have uploaded a file larger than 1GB. If you limited the share settings for the file, you can share it with specific staff members right away, or you can wait until your team members need the file for their engagement work. For troubleshooting information, see Java applet is not loading.

Linking a Google doc

If your team uses file storage services like Google Drive or Microsoft OneDrive to collaborate on some aspects of your engagement work, you can create links to these online files in Cloud. In order to do this, you must have linked the relevant account to your personal profile - for more information, see Integrate productivity tools with Cloud.

To link a Google doc:

  1. Ensure that you have the Editor role or equivalent rights for the client entity.

  2. From the Cloud menu, select Files.

  3. Select New, then choose Google Drive | Existing Google File.

  4. In the Reference dialog, paste the sharing link for the Google file and enter a description for the file.

    For more information about link sharing for Google documents, see Google's support.

  5. Select Save.

You have added the existing Google doc to the Files page for your client entity. You can link all the Google files that are relevant for your client engagement in this way.
