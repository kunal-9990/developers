Tracking incomplete and unreviewed documents

You can track incomplete and unreviewed documents by using workflows. You can create a workflow with a completed and reviewed workstage for any uploaded file, Google file or SharePoint file to help keep track of how many documents are still incomplete and unreviewed.

Een nieuwe workflow aanmaken:

  1. U heeft de Beheerdersrol nodig.

  2. Klik op Instellingen in het Cloudmenu.

  3. Ga naar Personalisering | Beheer Workflow.

  4. Bij het item waar u een workflow wilt aanmaken klikt u op (Niet toegekend) | Workflow aanmaken.

  5. Voer een Naam voor de nieuwe workflow in en klik op Toevoegen () zodat u de verschillende fases kunt aanmaken.

  6. U kunt de kleur ( ) van de fase nog aanpassen en met het icoon ( ) kunt u de volgorde van de fases aanpassen.

  7. Klik op OK.

To modify an existing workflow:

  1. Ensure you have the Admin role.

  2. Go to Settings | Customization | Workflow Management.

  3. Select Manage Workflows.

  4. In the Manage Workflows dialog, select an applicable workflow stage and select Edit.

  5. Make your changes to the workflow and select OK.

Delays

If you are using the CaseWare Audit template, you can use CaseWare Q to automatically feed when deliverables are completed from Working Papers to the Cloud. CaseWare Q has a user friendly interface that displays how many due deliverables are overdue and due this week.

Selecting the counter will provide you more information on which deliverables are overdue or due this week.

With Q-enabled files, you get a more comprehensive view of your engagement file. Opening the Engagement file details from the Files page, you can obtain important metrics to ensure you are providing efficient and effective engagements.

Instead of adding the deliverables in as tasks, CaseWare Q provides the most common audit deliverables from our groundbreaking Audit template. When the Deliverable is signed-off as completed and reviewed in Working Papers, the Deliverable is automatically marked as complete. Any deliverable that is overdue will display in red.
