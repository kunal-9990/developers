What's new - Analytics.AI beta release

CaseWare Analytics.AI is a risk-based transaction analysis tool that auditors can use to scan large transaction sets and find exceptions that might warrant further investigation.

See Analytics.AI workflow for more information.

Beta release known issues and limitations

Data Completeness has an issue in the Transactions column where values are not retained. This will be fixed shortly.

Import Transactions page:

  • Posting Status must be mapped for transactions to display in Data Completeness.

  • All Core mapping fields must be mapped. Additionally:

    • Under Additional Fields, Entered By and Entry Responsible Person must also be mapped for the tests to run. If you do not have these columns in your source, you can work around it by creating dummy columns with a value and mapping to those fields. We will be fixing this shortly.

    • The two Account Master fields Account Main ID and Account Main Description at the bottom of the Import page must be mapped for the Transaction drill-down to function correctly. You can map Account Main Id to the same field you mapped Account Id. For Account Main Description map to an available column (even if it is a duplicate mapping), or create a dummy column with data.

  • Entry Id must be entirely numeric.

  • Entered Date must be a string in the format of yyyy-MM-dd HH:mm:ss.

  • Currently, there is little feedback for error handling during the import process. We are working to address this very soon

Influencer changes have no effect. We are working to fix this.

Configuration page:

  • Setting a Risk Score to 0 will produce unexpected results. As a workaround, deselect the test if you do not want it as part of your test run.
  • Negative risk scores will produce unexpected results. Do not enter negative values in the Risk Score.
