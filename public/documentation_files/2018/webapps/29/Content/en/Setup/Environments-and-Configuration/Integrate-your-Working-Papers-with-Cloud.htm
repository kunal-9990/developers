Integrate Working Papers with Cloud

This feature is only available with CaseWare Working Papers.

Learn more about Working Papers.

Integrating Working Papers with CaseWare Cloud provides additional functionality. Cloud can act as a file management and storage center, as well as provide expanded options to track and complete your engagement.

When Cloud is integrated with Working Papers in a terminal server environment, all staff that launch Cloud are able to sign in with their own credentials.

To integrate Working Papers with Cloud:

  1. Launch Working Papers.

  2. On the File menu, select Options | CaseWare Cloud.

  3. Ensure Integrate with CaseWare Cloud is selected. In the Server field, enter the URL of your organizationâ€™s CaseWare Cloud site. You do not need to enter the web protocol (http:// or https://). Click OK.

  4. A CaseWare Cloud Login prompt is displayed. Enter your User name and Password, then select OK.

Your copy of Working Papers is now integrated with Cloud. If you are operating Working Papers in a terminal server environment, you can now set a default Cloud URL for your organization to help save time when signing in.

Set a default Cloud URL

When working in a terminal server environment, staff members will need to enter their credentials, including a Cloud URL, each time they open Working Papers. To save time during sign in, you can set a default Cloud URL that is automatically added when Working Papers is opened.

To set the default Cloud URL in Working Papers:

The registry key is located in:

  • HKEY_CURRENT_USER\SOFTWARE\CaseWare International\Working Papers\2016.00\Core

  1. Right-click the Server key and select Modify....

  2. In the Value data: field, enter your Cloud URL. Do not include the web protocol (http:// or https://) in the URL.

  3. Click OK to apply your changes.

A default Cloud URL is set.
