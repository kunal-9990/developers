Set up dynamic tables

Tables refer to the different schedules that you can include in your financial statements document.

To add a table in a financial statements area, select Add (), then choose Dynamic Table. An empty table is added in the area with a default title Dynamic Table. You can change the default title by selecting it and add a new title in the field.

The following steps summarize the recommended workflow for setting up dynamic tables. You can use this workflow for guidance.

  • Add rows

  • Add columns

  • Add a note reference in rows and columns

  • Allow users to add input rows in the table area without accessing the tableâ€™s edit mode

  • Add a page break in the table

  • Copy the table to quickly create another one with similar setup

Add rows

Add rows to define the data in your table. You can also add rows to insert white space to improve the table readability.

To add a row:

  1. Select the table.

  2. Select More actions (), then choose Edit Dynamic Table.

    3. The table opens in edit mode.

  3. Go to where you want to add a row.

  4. Select Add () at the left border of the table.

  5. Select one of the following row types:

    • Trial balance data: to select the row data from the trial balance data. You can insert an account or group. If the group has subgroups, they are inserted into the table as well.

    • Input row: to add a text field row. You can enter custom text.

    • Spacing row: to add white space into your dynamic table. You can add white space to make your content easier to read.

    • Page break: to add a page break between table rows.

    • Custom group: to add a custom group row. Custom group rows insert a text field so you can enter a title or a brief description for your group.

  6. Select SAVE or SAVE & CLOSE.

As you work on the table, you might need to rearrange rows or delete some of them.

To move a row:

  1. Select the table.

  2. Select More actions (), then choose Edit Dynamic Table.

    3. The table opens in edit mode.

  3. Locate the row that you want to move, then select the Move ().

  4. Drag the row then drop it in the desired location.

    5. A grid displays in the table to mark the borders of each row. Valid places to move your row have dark blue grid lines while invalid places have gray grid lines. Use the grid for guidance as you drag and drop.

To delete a row:

  1. Select the table.

  2. Select More actions (), then choose Edit Dynamic Table.

    3. The table opens in edit mode.

  3. Select the check box at the right border of the row.

  4. Select Delete () .

Add columns

Add columns to define the data values in your table. You can also remove rows if needed.

To add a column:

  1. Select the table.

  2. Select More actions (), then choose Edit Dynamic Table.

    3. The table opens in edit mode.

  3. Go to where you want to add a column.

  4. Select Add () at the upper table border.

    5. A column is added to the table and the column settings display on the page.

To set up the column data:

  1. Select the column header.

  2. Modify the following settings as needed:

Type

Choose a column type. The following columns have predefined values:

  • Prior year - prior year balance
  • Current year - current year balance
  • Current period - balance for the current period (for interim engagements)
  • Comparative period - balance for the comparative period
  • Amount variance - difference between prior year balance and current year balance
  • Percent variance - percentage difference between prior year balance and current year balance

You can enter custom values for the following columns:

  • Input - enter text, numbers, currency values or percentages.
  • Custom calculation - enter a calculation using the calculation editor. The calculation is applied to all column cells.
  • Note reference - add references to existing notes in the financial statements.
  • Annotation - add annotations column. Select Add () in each cell in the column to add annotations.
Source Type

Select the data source for the column:

  • Trial balance - trial balance data displays in the column
  • Sum of transactions - the sum of transaction for the group displays in the column
  • Number of transactions - the number of transactions for the group displays in the column

Note:Sum of transactions and Number of transactions are only functional if you enable the Data page (new design of Trial Balance page) so users can import transactions.
Carry forward action

Select the carry forward action:

  • None - column data doesnâ€™t change in the next period file
  • Move data - column data is moved to the next period file
  • Clear content - column is empty and users need to add the data for the next period file

Note:Carry forward action is only available if you select Trial balance in Source Type.

Heading

You can insert Dynamic Text or formula (fx) for the header. You can also type simple text in the Output field or combine simple text with dynamic text and formula.
Dimension

You can select a dimension to set the column to only display data for that dimension.

Width

Select a width for your column. The width options for all columns are Narrow and Standard, except for note reference, annotation and description columns, where the width options are Small, Medium and Large.

Format

For input or custom calculation columns, you can choose one of the following formats for your cell values:

  • Currency
  • Number
  • Percent
  • Text

If you select Number or Percent, you can also choose the number of decimal places.

Display options

For some column types and formats, you can select one or more display options:

  • Expand to fill the width of the page - sets the column width so the dynamic table fits the full width of the page.

    • Note:This setting is enabled on the Description column by default.

  • Display zero balances - shows accounts with zero balances in the column.

  • Display opposite sign for balances - displays the opposite sign (negative or positive) for the values in the column.

  • Exclude from totals - excludes the values in this column from the calculation in the account group Total row.

  • Display symbol in heading - displays the currency symbol or percentage sign at the top of the column.

  • Show year in heading - adds the year to the heading of Current year or Prior year columns.

  • Use non-consolidated data - if this is a consolidated engagement, selecting this option removes consolidated data from the column and displays the non-consolidated data only.

Exclude / Include ( )

Select this icon to hide or show the column in the table's edit mode and in the financial statement document.

Delete ()

Select this icon to delete the column.

  1. Select DONE.

  2. Select SAVE or SAVE & CLOSE.

To edit dynamic table descriptions:

  1. Go to the Description column.

  2. Select the description that you want to edit.

  3. Edit the name as desired. Your changes are immediately applied to the table.

    4. Note: If there is trial balance data in the dynamic table, the Revert To icon () displays next to a modified description. Select Revert To () to revert to the original trial balance description.

Add a note reference in rows and columns

A note reference is a link to an existing note in the financial statement. You can add note references in table rows in the financial statement. For example, to link an account or a group to an existing note that holds related details or information.

To add a note reference in a row:

  1. Select the table.

    Icons for table options display on the right.

  2. Select More actions (), then choose Edit Dynamic Table.

    3. The table opens in edit mode.

  3. Go to the row where you want to add a note reference.

  4. Select the Add () button above the row.

  5. Choose Commentary row.

    6. A note reference is added right above the row.

  6. Select (Note#) to display a list the existing notes in the financial statements document.

  7. Select one or more notes from the list to reference it in the row.

    8. You can also select Show hidden notes to reference a hidden note .

  8. Select SAVE or SAVE & CLOSE.

    9. The number of the referenced note displays next to the row name.

You can also use the above method to change a note reference or you can change it directly without opening the table in edit mode.

To change a note reference without opening the table in edit mode:

  1. Select the note reference that you want to change.

    2. A list the existing notes in the financial statement displays

  2. Select one or more notes from the list to reference in the row.

    3. You can also select Show hidden notes to reference hidden notes.

To add a note reference in a column:

  1. Add a column in the table.

    1. Select the table.

      Icons for table options display on the right.

    2. Select More actions (), then choose Edit Dynamic Table.

      3. The table opens in edit mode.

    3. Go to where you want to add a column.

    4. Select Add () at the upper table border.

      5. A column is added to the table and the column settings display on the page.

  2. Select the header of the new column and change the column type to Note reference.

  3. Enter a header and specify the desired width.

  4. Select Done.

    5. The new column is added in the table with empty references (#).

  5. Specify the note references for each row.

    1. Select an empty reference (#) to display a list the existing notes in the financial statements document.

    2. Select one or more notes from the list.

      3. You can also select Show excluded notes to reference excluded notes.

  6. Select SAVE or SAVE & CLOSE.

Allow users to add input rows directly in the table area

You can allow users to add input rows (custom rows) in a dynamic table directly without opening the table in edit mode. This simplifies the process of adding input rows for end users, especially in cases where adding custom rows in a table happens frequently.

For example, if you know that in a particular place in the table, users add labor costs on multiple lines, so they add input rows. You donâ€™t predefine the input rows for them, because you donâ€™t know how many rows theyâ€™d need to add. If this case happens frequently, it can be overwhelming for users to go to the table set up screen (edit mode) each time they need to add extra rows in the table.

Note: This option can only be enabled in specific places in the table. You must know the specific place or places in the table where you want to allow them to add input rows outside of edit mode.

To enable adding input rows directly in the table area:

  1. Select the table.

    Icons for table options display on the right.

  2. Select More actions (), then choose Edit Dynamic Table.

    3. The table opens in edit mode.

  3. Locate where you want to allow users to add input rows, then select the Add () | Custom group at the left border of the table.

    4. Note: Adding a custom group marks the place in the table where you want to allow users to add input rows outside edit mode.

    A new group row is added in the table with a default title Title. You can select the default title to change it.

    A Total row is also added in the table. It holds the total value for the balances in the input rows added by the user.

    If you donâ€™t want the group row or the total row to show on the table, select the row then select the eye icon () to hide it. In this case, itâ€™s recommended to also select Settings () | Show excluded items to keep track of all existing rows in the table as youâ€™re working on it.

  4. Select the new custom group.

    5. The Enable Input Rows icon () displays on the top left corner.

  5. Select Enable Input Rows ().

  6. Select SAVE & CLOSE.

The Add Row () button is now available in the exact place in the table where you want to allow users to add input rows.

To disable this option for users, select the group you added in Step 3, the select ().

Add a page break in the table

You can add a page break inside a table in the document to better present the table's information. For example, you can insert a page break inside a table to have similar or related information show on the same page rather than being spread over two pages.

To add a page break in a table:

  1. Select the desired table.

    2. Icons for table options display on the right.

  2. Select the More actions ()

  3. Choose Edit Dynamic Table from the list of actions.

    4. The table opens in edit mode.

  4. Locate where you want to add a page break.

  5. Select the Add() | Page Break at the left border of the table.

  6. Select SAVE or SAVE & CLOSE.

Change the table format

You can change the dynamic table format to support the International Financial Reporting Standards (IFRS) and the Statement of Changes In Equity (SOCIE). The new format flips the table so you can have dimensions as rows and trial balance groups as columns.

Note: When you change the format, existing setup and information in the table is lost and you need to redo it in the new format.

To switch to IFRS/SOCIE table:

  1. Select the desired table.

    2. Icons for table options display on the right.

  2. Select More actions () | Edit Dynamic Table.

    3. The table opens in edit mode.

  3. Select Settings (), then select IFRS/SOCIE under Format.

    4. A confirmation prompt displays to notify you that switching the table format creates a new table and all existing setup and data will be lost.

  4. Select Create a new table.

  5. Setup the table rows and columns as desired.

The following row and column options are now available in the table:

Rows
  • Dimension row - select a dimension to set the column to only display data for that dimension.
  • Input row - adds a text field row. You can enter custom text.
  • Commentary row - adds a commentary row. By default it inserts a reference to existing notes in the financial statements. You can change the default by deleting the note reference and enter a comment for the row.
  • Spacing row - adds white space into your dynamic table. You can add white space to make your content easier to read.
  • Page break - adds a page break between table rows.
  • Custom group - adds a custom group row. Custom group rows insert a text field (to add a title or a brief description for the group) and a total row.

Columns
  • Trial balance -  adds a trial balance date column. You can select an account or group to source the column data from. If the group has subgroups, they are inserted into the table as well.
  • Input -  adds a text field column. You can enter custom text.
  • Custom calculation - adds a custom calculation column where you can enter a calculation using the calculation editor. The calculation is applied to all column cells.
  • Note reference - adds a column of references to existing notes in the financial statements.
  • Annotation -  adds an annotation column. Select Add () in each cell in the column to add annotations.

Copy a table

You can copy an existing table and paste it anywhere in the document. Once you copy a table, it stays in the clipboard and you can paste it as many times as you want until you copy a different table to clipboard.

To copy a dynamic table:

  1. Select the dynamic table that you want to copy.

    Icons for table options display on the right.

  2. Select More actions ()| Copy Dynamic Table.

  3. Select where you want to add the copied table, then select Add()| Paste Copied Dynamic Table.
