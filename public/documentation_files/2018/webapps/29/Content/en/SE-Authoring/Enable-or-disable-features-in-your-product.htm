Enable or disable features in your product

You can enable some features in your product using the API. To enable a feature, add the property under each feature in the product.json file. To disable a feature, delete the property and its value.

See tutorial-product-features for the full list of product features and the required property setting for each feature.

Areas

Allows you to summarize financial statement groups at a higher level for risk identification and assessment. For more information, see Add areas, phases and cycles.

Automapping

Automatically maps sections of data files on import.

See Imports for more information on the automap functionality.

Checklist Guidance

Enables guidance for checklists.

For more information, see Add document guidance.

Cycles (business cycles)

Used to further organize the groups in the Trial Balance.

For more information, see Add areas, phases and cycles.

Dimensions

Dimensions are properties that can be assigned to accounts in an engagement. Each account can be assigned one or more dimensions.

For more information, see Add dimensions.

Engagement Properties

Prompt Displays the engagement properties each time users open a new engagement. For more information, see Set the engagement properties.

Issues

Allows users to add issues.

For more information, see Add issues to an engagement file.

Materiality

Enable materiality in your product. An accepted misstatement (due to omission or error) is considered material. Materiality is the accepted threshold (error margin) for misstatements.

For more information, see Set up Materiality and Summary of Misstatements.

Prior Years

Displays the prior five years (not just one) in the Trial Balance.

For more information, see Define groupings.

Query

Allow users to create queries.

For more information, see Create a client request.

Risk Assessment

To enable the Risk Assessment report in your product.

For more information, see Set up Risk, Control and RMM.

 

Roll Forward

Enables carry forward. The product.json file in the API needs to be modified to include the previous and next years when they exist. See Enabling carry forward for external products for the detailed instructions.

For more information on carry forward, see Carry forward an engagement.

Second Reviewer

Enables a second reviewer slot in the engagement file details in Cloud.

To open the engagement file details in Cloud, select the file. A panel displays on the right side showing the file details.

Statement

Enables extra statements in the financial statements document. For example, the statement of Functional Expenses.

For more information, see Customize financial statements.

Trial Balance

Enable the Trial Balance tab in the engagement.

For more information, see Define groupings.

User Defined Financials Groups

Allows users to add their own groups.

For more information, see Add a group.

XBRL

Enables XBRL functionality throughout the product. A conversion support file and a pointer to it must be added in the API.

See Starting with XBRL for the detailed instructions.

Review Tools

Disables the Review Tools throughout the product for users.

For more information, see Review prepared documents with review tools.

Engagement Search

Disables the search field at the Document Map.

For more information, see Locate and access documents in the engagement files.

Interim Reporting Period

Enables interim reporting.

For more information, see Create an engagement for an interim reporting period.

Workflow

To enable read-only workflow feature or to hide the workflow status at the engagement properties dialog.

For more information, see Creating workflows.

Reimport

Allows update of account name and number during trial balance reimport.
