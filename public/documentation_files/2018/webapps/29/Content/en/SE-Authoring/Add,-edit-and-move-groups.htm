Add, edit and move groups

You can edit the properties or existing groups or rearrange them. You can also add groups.

Add groups

You can manually add groups to define a grouping.

To add a group:

  1. Go to the Trial Balance page, then select Add () | Group.

    2. The Group Properties dialog displays.

  2. Set the properties for the new group.

    3. The following table describes the available group properties:

Group property

Description

Number

Enter a unique mapping number. If you have already imported groups, the mapping number must follow the existing mapping mask structure.

Consistent Name

Enter a description for the map number. This description appears as the preliminary wording in the financial statements.

Contrary Name

If applicable, enter a description for group entries that are opposite to the group’s normal sign. For example, "Bank Overdraft" will appear instead of "Bank" when the account balance is a credit.

Inconsistent Name

If applicable, enter an alternative description for the group when the current and prior year have opposite signs.

Note: This name takes precedence over the consistent name and contrary name regardless of the group balances in the Trial balance.

Sign

Select the applicable sign (debit or credit).

Type

Select the appropriate statement type for this group (Balance Sheet, Income Statement of Comprehensive Income).

Formula

If applicable, enter an AOCI formula to calculate the values for this group. Once your user imports the trial balance, this group will display an amount derived from the balances in other groups.

Here is a sample AOCI formula:

Tagbal("1.234",perspec({yearOffset}),{type},{code}) - where 1.234 is the OCI group in the Income Statement section

Breakdown by dimension

If applicable, select a dimension category for the group. This enables users to filter the trial balance so that they can view balance information by dimension.

Cycle

Select a business cycle for the group. When you assign a group to a business cycle, items in that group will automatically be shown under this cycle in the Risk Assessment document, making it easier to audit by business cycle.

Link to secondary group

Select a group in the secondary grouping to link the financial group to it. This enables you to automatically assign accounts to groups in the secondary structure based on their initial group assignment.

Properties

  • breakdown = visible - Enter this property for an account group that requires account subgroups expand by default in the financial statements. For example, Property, plant and equipment displays the subgroups Property, plant and equipment and Property plant and equipment accumulated depreciation below it.

  • It can be collapsed in Financial Statements by selecting the Collapse group icon beside the group.

  • breakdown = hidden - Enter this property for account groups that require account subgroups to collapse by default in the financial statements so that the unnecessary details are hidden.

    You can expand the details in the Financial Statements by selecting the Expand group icon beside the group.  

  • breakdown = none - Enter this property to prevent sub-account groups from being displayed for this account group.

  • highLevelView = true - Enter this property for any account groups that should be collapsed when the user selects the Collapse button in the trial balance.

  • financialClassification = netIncome - Enter this property to treat the group as net income.

Allow account assignment

Select to allow accounts to be assigned to this group.

Reassign target for contrary accounts

If applicable, select a group (“reassign target”) for accounts with a contrary balance. If an account would be automatically assigned to this group but it does not match the selected sign (debit or credit) for this group, it will be automatically assigned to the reassign target.

For example, negative Asset --> positive Liability

Classification

Type or select a classification from a master chart of accounts. This master list of classifications allows direct automapping from an accounting package by matching the classification applied to the group to the account-type in the accounting package.

Adjust preference of group

Weight is used to automap accounts to a group based on a scale. For example, groups with a positive weight will be preferred over groups without a weight assigned.

Account balance restrictions

Specify the types of accounts that can be assigned based on their balance:

  • accounts with any balance to be assigned
  • only accounts with a debit (positive) balance to be assigned
  • only accounts with a credit (negative) balance to be assigned

  1. Select Save to add the group.

Edit groups

You can edit the properties for any group in the structure.

To edit a group:

  1. Go to the Trial Balance page, then choose the grouping structure from the drop-down list at the top.

    2. You can choose to edit for Financials , Tax Code or Chart of Accounts grouping.

  2. Select the group you want to edit, then select Edit ().

    3. The Group Properties dialog displays.

  3. Edit the group properties as needed.

    4. See the group properties table for a detailed description of each property.

  4. Once you’re done, select Save.

Move groups

You can change the structure of the grouping by moving groups.

To move a group:

  1. Go to the Trial Balance page.

  2. Select the group that you want to move, then drag and drop into the desired place in the structure.

    3. A rectangular grid displays in the destination place to identify the target spot for the group.

  3. To convert the group to a subgroup under a specific parent group, drag it and drop in the target parent group.

    The target parent is marked with a green background.
