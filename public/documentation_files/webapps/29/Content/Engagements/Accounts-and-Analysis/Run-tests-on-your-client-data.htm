This feature is only available with CaseWareCloud Analytics, part of the CaseWare Cloud suite.

Learn more about Cloud.

In Cloud, client data can be tested from the Analytics app. Analytics enables you to test financial client data from a centralized location. You can test for errors and irregularities in your file, locate duplicates, and ensure that clients adhere to standards through convenient reports and charts. Analytics integrates with CaseWare Cloud to provide quick and easy access to any data that you already manage in Cloud.

Getting started

If you are just getting started with Analytics, your first step will be to create an Analytics engagement. Analytics engagements can be used to organize the client data you upload to Cloud, with one engagement representing one time period per client.

To create an Analytics engagement:

  1. Ensure that you have the Editor role or equivalent privileges.

  2. From the Cloud menu, select Analytics.

  3. Select New.


  4. Specify an Entity, then complete the remaining fields.

  5. Select Save.

The Analytics engagement is created. You can see your new engagement on the Analytics page.

Once youâ€™ve created an Analytics engagement, you can begin to import your client data. The way you import this data depends on what accounting software the data was created in. If the data was created in one of the following software packages, you can use the CaseWare Transformer Tool to convert your client files to a formatted .zip package, then import the .zip file into Analytics:

  • AuditXML 3.x
  • MyOB AccountRight Live 2013 - 2016
  • Pastel Partner 12.1-14.0
  • QuickBooks CA 2012 - 2017
  • QuickBooks US 2012 - 2017
  • Sage 50 Accounting (Peachtree - WIN) 2012 - 2017

Otherwise, you will need to import your client data in .csv file format. The .csv files should follow specific formatting guidelines to ensure successful import and testing.
