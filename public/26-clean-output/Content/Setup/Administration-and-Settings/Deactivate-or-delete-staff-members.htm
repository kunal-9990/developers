Deactivate or delete staff members

There are several considerations when a staff member leaves your organization either temporarily or permanently. As an administrator, you will need to decide if it is more appropriate to deactivate or delete their Cloud account, and you will need to reassign their work and transfer ownership of any Cloud documents that are associated with them.

Deactivating staff or contacts

Deactivating a user prevents them from logging into Cloud, but keeps their profile in the system and preserves their files, activities, and discussions. You may want to deactivate a user if they are going on extended leave.

To deactivate a staff member or contact:

  1. Ensure that you have the Staff Admin role or equivalent privileges.

  2. Go to the Staff or Contacts page.

  3. Select the user whose status you want to change, then select edit ().

  4. From the Status drop-down, select Inactive.

  5. Select Save.

You have deactivated the user. They will no longer be able to long into Cloud.

Deleting staff or contacts

Deleting a user removes them from the system, and prevents them from logging into Cloud, as well as accessing any SmartSync-enabled Working Papers files. Their files, activities, and discussions will remain in the system. You may want to delete a user when their relationship with your organization ends.

To delete a staff member or contact:

  1. Ensure that you have the Admin role for Contacts or Staff, or the equivalent privileges.

  2. From the Cloud menu, select Staff or Contacts.

  3. Select the user you want to delete, then select ellipsis () | Delete.

  4. In the confirmation dialog, select Yes.

You have deleted the user.

Deleting staff members synced in CaseWare Time desktop

If your organization uses CaseWare Time with Cloud integration enabled, staff must be deleted from the Time app. After Time users are deleted, they will not be able to log back in and any client assignments for that user will be removed.

To delete a staff member in Time:

  1. Ensure that you have the Staff: Edit permission in Time and the Staff Admin role or equivalent permissions in Cloud.

  2. Ensure that the user you want to delete does not have any of the following responsibilities, or else you will not be able to delete them:

    • Current work-in-progress assignments.

    • Set as Partner or Manager of any group, client, staff member, department or assign area.

    • Set as In Charge or Team Leader of any project template, subproject template, client project or client subproject.

    • Assigned to any issue, client, or group.

    • Is the sole staff member with Administrator privileges.

  3. In Time, select Staff | Setup.

  4. Select or enter the Staff No. of the user you want to delete.

  5. Click Delete.

Reassigning a staff member’s work

If you’ve just deleted a Cloud staff user that was associated with file requests or tasks, you’ll need to reassign these activities to a new user. File requests are reassigned using the file’s share settings, while tasks are reassigned by editing the task directly.

To re-assign a file request:

  1. Ensure you have the Entity Access role or equivalent rights to the entity.

  2. From the Cloud menu, select Activities.

  3. From the Activities drop-down, select File Requests.

  4. Select the file request you want to reassign. From the file’s menu, select Share ().

  5. Select the user you want to reassign the request to, then select Assign roles to (#) and the appropriate roles.

  6. Select Share.

You have reassigned the file request.

To reassign a task:

  1. Ensure you have the Entity Access role or equivalent rights to the entity.

  2. From the Cloud menu, select Activities.

  3. From the Activities drop-down, select Tasks.

  4. Select the task you want to reassign, then select Edit () from its menu.

  5. In the Assignees field, clear any users you want to remove from the task and add the users you want to reassign the task to.

  6. Select Save to apply your changes.

You have reassigned the user's task.

Transferring cloud document ownership

If the staff being removed from Cloud has Google Docs or SharePoint files associated with their account, you can transfer ownership of these files to another user with a Google Docs or SharePoint account to ensure that your organization doesn’t lose access to any important materials.

To transfer document ownership:

  1. Ensure you have the Owner role or equivalent privileges.

  2. Select the file whose ownership you want to transfer.

  3. Select ellipsis () | Transfer.

  4. A dialog opens listing all applicable users. Select the user you want to transfer file ownership to, then select OK.

You have transferred the document's ownership.
