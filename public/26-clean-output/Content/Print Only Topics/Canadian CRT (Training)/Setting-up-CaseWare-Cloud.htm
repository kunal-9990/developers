Setting up CaseWare Cloud

Before you can start work on any Review and Compilation – Canada engagements, you need to set up the new CaseWare Cloud portal for your organization. CaseWare offers a Getting Started guide that can walk you through the common setup tasks, but for the purposes of this training, there are three things that you will need to do:

  • set up workspaces in CaseWare Cloud for each of your clients
  • create accounts for your staff members
  • create accounts for your clients

This chapter will walk you through the process for completing these setup steps. Once these steps are complete, you will be ready to create a Review and Compilation – Canada engagement.

Create a new entity

An entity is a workspace that you can use to organize and collaborate on your engagement files from a single location for a client. When your organization is first beginning to use Cloud, you will need to create a new entity for each client.

Note: The instructions below explain the process for adding an individual client entity. You can also import a list of your clients in order to create all of your client entities at once. For more information, refer to CaseWare’s online help.

Create accounts for your staff members

Once you have created entities for each of your clients, you will be ready to grant access to Cloud to other staff members in your organization. You can create a new account for each staff member who will need to collaborate on your engagements.

Note: The instructions below explain the process for adding individual staff members. You can also import a list of your staff members in order to create all of their user profiles at once. For more information, refer to CaseWare’s online help.

From the Cloud menu, select Staff. This takes you to the Staff page.

Select New, and then under Add, select Staff. The Add Staff dialog is displayed. Just as you saw when creating an entity, there are different property groups where you will enter information about the new account.

General

Enter an email address and a first and last name for the new staff member. The email address will be the username for their account, and they will log into Cloud with it.

The other fields on this page are optional.

Password

You can choose how the password for the new account will be set:

  • Set by User - the staff member will set their own password when they first log in. Cloud sends an email with instructions on how to log in to the portal.
  • Auto Generated - a random password is chosen by Cloud. Cloud does not send an email to the staff member, so you will need to provide them with the login page URL, their user ID, and their password.
  • Set - you choose a password for the staff member. Again, Cloud does not send an email, so you will need to provide them with their login information.

Leave the password option as Set by User.

Groups

You can assign the staff member’s account to one or more staff groups. If you create groups in Cloud, you can specify permissions and access levels for members of that group, and then any accounts that you add as group members automatically inherit those settings.

CaseWare Cloud only includes the All Staff staff group by default, and any staff members are automatically included in the group, so you won’t have to do anything in this section right now.

System-Wide Roles

When you grant one or more system-wide roles to a staff member, they gain a set of access rights that are applied to all of the content in Cloud. For example, if you grant the system-wide Viewer role to your staff member, they can see all of the entities in your organization and can view all files and engagements in those entities. System-wide roles are the easiest way to set the access rights for staff members in your organization.

Grant this staff member the system-wide Editor role. This will enable them to view all of the content in Cloud, and it will give them the ability to make changes while working on that content.

Apps

You can grant access to apps on Cloud to your staff member. You can grant the staff member the ability to view engagements for an app and the ability to create new engagements or files for it.

Note that by default, the All Staff group includes the ability to view and create engagements for all of the Cloud apps. This simplifies access, but if you needed to control access to some apps more tightly, you could remove some of those permissions from the global group. This would enable you to pick and choose which staff members should be able to work with certain apps.

Standard Hours

The settings for Staff Standard Hours are only applicable to your organization if you use the CaseWareCloud Time app to handle time and expense entries. Leave this section set to Organization standard hours.

Once you have finished entering all of the information for your staff member’s new account, select Save. They will receive a welcome email from Cloud notifying them that their account has been created and instructing them on how to log in.

Create an account for your client

One of the major benefits of Compilation and Review is that you can collaborate directly with your clients on engagements. In order for your clients to access their engagement file, you need to set up a contact account for them on Cloud.

From the Cloud menu, select Contacts. This takes you to the Contacts page.

Select New, and then under Add, select Contact. The Add Contact dialog is displayed.

General

Enter a first and last name and an email address for your client contact. In the Entity field, select the client entity that you created earlier. Once you have selected the client entity, select the Contact - Entity Collaborator role to set the access level for this account. The entity collaborator role enables your client to create discussions, tasks, and file requests on Cloud, and it also enables them to collaborate on engagements for Compilation and Review.

Password

The same password options that you had when creating your staff accounts are also available here. Leave the password option as Set by User.

Groups

Contact groups work the same way that staff groups do on Cloud. The only contact group in Cloud by default is All Contacts, and your client will automatically be part of that group.

Apps

Just like the All Staff group, the All Contacts group automatically grants client contacts the ability to view engagements or files for all app types in Cloud. Review the permissions and make sure that your contact can view Review and Compilation – Canada engagements.

Once you have finished entering all of the information for your client contact’s account, select Save. They will receive a welcome email from Cloud notifying them that their account has been created and instructing them on how to log in.
