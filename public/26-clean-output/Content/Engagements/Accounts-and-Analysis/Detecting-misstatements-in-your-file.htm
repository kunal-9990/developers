Testing for misstatements or fraud in your file

Analytics offers several tests specifically designed to look for misstatements, errors, or possible fraud in your client file to help assess risk and ensure that reported information conforms with accounting standards. These tests produce reports, which you can then save to your Analytics engagement, download to your computer, or export directly to a Working Papers engagement.

For example, you can run the Outlier detection test from any Analytics data file categorized as a journal or ledger type (such as a General Ledger or Sales file) to look for any values in your file that is outside the normal scope of the file.

The Benfordâ€™s law test, which is available for all .csv format data files, applies the principle that in many sets of numbers (including amounts), some digits and digit combinations tend to occur more than others. The test examines the data to see how closely it conforms to this pattern.

Finally, the tests Entries posted at unusual times and Entries posted on unusual days, available for all journal and ledger, identify entries posted outside the normal scope of operations.

Running Analytics tests

Please note that to run most Analytics tests on .csv format data files, you will need to have mapped the file. For more on mapping data files, see Import client's financial data from a CSV file for analysis .

To run an Analytics test:

  1. Ensure that you have the Editor role on the file or equivalent privileges.

  2. From the Cloud menu, select Analytics.

  3. Select your Analytics engagement, then select the appropriate data file from the Data Files pane on the left.

  4. Select the Tests tab to show a set of test categories.

  5. Select the arrow next to a test to run it. A green arrow () indicates that you can run the test without providing any parameters or additional mapping. If a test displays a yellow () or dark gray () arrow, you will be asked to provide test parameters or map additional columns before you can run the test.

  6. On the test results page, select Save to keep a copy of the report to your engagement. The report appears as a child of the original data file.

You have generated the Analytics test result and saved it to Cloud.

If required, you can download the test result to your computer or directly to Working Papers to include in your engagement file.

To download an Analytics report to your computer:

  1. Ensure that you have the Editor role on the file or equivalent privileges.

  2. From the Cloud menu, select Analytics and then select the appropriate Analytics engagement.

  3. Next to your saved report, select ellipsis () | Download and select the appropriate download option. When a graph is available, you can download it in .png format.

You have downloaded the test result to your computer in the selected format.

To download an Analytics report to a Working Papers engagement:

  1. Ensure you have the Editor role on the file or equivalent privileges.

  2. In Working Papers, click File | Options | CaseWare Cloud. Sign in with your Cloud credentials.

  3. Select Cloud | Engagement, Entity, or Files to open Cloud in your Working Papers browser window.

  4. In the browser window, select Analytics from the Cloud menu.

  5. Next to your saved report, select ellipsis () | Download and select the appropriate download option. When a graph is available, you can download it in .png format.

  6. In the File Download dialog, click Add.

  7. In the New Document Link dialog, specify the appropriate settings for the document and click OK.

You have added the Analytics report to your Working Papers Document Manager.
