Review notes can be added to an engagement workspace (entity) or to documents, checklists and other engagement artifacts in an engagement file (online and desktop). Review notes can signal to the engagement team that additional work is required on a document or that specific tasks must be completed for an engagement.

In CaseWare Cloud, review notes are referred to as tasks and are added to the engagement workspace (entities). In an engagement file (such as Working Papers), review notes are referred to as issues.

Best Practice: It's better to create an issue from within the engagement file so that your team can have context as to which document, checklist, line or engagement requires attention. For general review notes, consider using the tasks feature in the engagement workspace.

Add review notes to a client workspace

Adding tasks to an engagement workspace enables you to define all the work that is required for a particular engagement. As the engagement team resolves tasks, you can track the engagement progress and assess when the expected completion date for the engagement will be.

To create a task:

  1. Ensure you have the Entity Access role or equivalent rights to the entity.

  2. From the Cloud menu, select Activities.

  3. From the Activities drop-down, select Tasks.

  4. Select New from its menu.

  5. Select an entity for this task and at a minimum complete the Subject line.

  6. (Optional) Select an assignee from your staff list and specify a due date.

  7. Select Post to create your task.

Review your assigned review notes

By applying a filter to the Activities page, you can view all open Tasks that are currently assigned to you.

To view tasks assigned to me:

  1. From the Cloud menu, select Activities.

  2. From the Activities drop-down, choose Tasks.

  3. Select Filter () | Assigned To | Me.

  4. To only view currently open tasks, select Filter () | Status | Open. This will add a secondary filter so that only open tasks assigned to you are displayed.

A list of your assigned tasks is displayed.

Add review notes to an engagement file (SE)

This section describes functionality only available with products on the CaseWareCloud SE platform.

Learn more about Smart Engagement products.

You can add issues to a specific document or the entire engagement file. The command is available from the ribbon menu at the top of the page.

To add an issue:

  1. From the top menu, select the Add Issue icon ().

  2. In the Issue editor, enter a description for your issue.

  3. Select an issue type from the drop-down menu at the bottom of the editor.

  4. Select Create Issue.

View a summary of review notes

All review notes that are outstanding, resolved or cleared are accessible on the Issues report. You can use this report to determine what issues currently require review, or to view the history of review on a document.

Note: Deleting a review note will not save it on the Issues report. Notes must be cleared to retain the history.

To view a summary report of issues and review notes:

From the top menu, select Issues. The Issues report is displayed.

All of the issues for your current engagement are displayed on this report. For each issue, you can see:

  • issue type

  • description of the issue

  • replies from engagement team members

  • status changes

  • current status

At the top of the report, issues are summarized by type. You can use the drop-down menus to filter the report by issue type and status.

The record of current and historical review notes is displayed. You can change the status of outstanding issues (Resolve, Clear, Delete) or edit them in this view.

Add review notes to an engagement file (Working Papers)

This section describes functionality only available with CaseWare Working Papers.

Learn more about Working Papers.

You can create a review note directly in Review on the Go. These review notes are automatically synchronized to the client file and visible to all engagement team members when they open the file.

To create and assign a review note:

  1. In the document being reviewed, select the Create an Issue button ()

  2. Complete the review note details.

  3. Select Save.

You have created a new issue. This new issue will be added to the client file and accessible from both Review on the Go and Working Papers.
