When completing review notes you may be required to collaborate with other staff and client contacts. Whether you're assigned a review note from an engagement file or the engagement workspace, you can easily ping and tag other people on the item to get clarification or additional client information.

Once complete, mark the task as completed or the issue as resolved.

In an engagement file you also have the option to clear the issue, thereby removing the notice while maintaining the note itself.

Best Practice: It's better to create an issue from within the engagement file so that your team can have context as to which document, checklist, line or engagement requires attention. For general review notes, consider using the tasks feature in the engagement workspace.

Complete review notes in a client workspace

When assigned tasks from the engagement workspace (entity), you can respond and collaborate with other staff and client contacts to complete the work. When you're finished simply change the state of the task to Completed.

Comment on an assigned review note

If you need more information or context to clear an issue adequately or to update your supervisor on your progress, you can discuss the task with its poster.

To comment on an activity:

  1. Ensure that you have the Viewer role on the activity or equivalent privileges.

  2. From the Cloud menu, select Activities.

  3. From the Activities menu, choose Discussions.

  4. In the Details pane, select Comment to add a comment to the activity.

  5. Select Post.

You have posted your comment. Any users you tagged are now subscribed to the task.

Close review notes

Once you’ve performed the assigned task, you can clear it in Cloud by changing its status from Open to Completed.

To clear review notes:

  1. From the Cloud menu, select Activities.

  2. From the Activities drop-down, choose Tasks.

  3. Select the Task you want to close.

  4. In the Status drop-down menu, select Completed.

You have updated the task’s status. Any users subscribed to the task will be notified.

Completing review notes in an engagement file (SE)

This section describes functionality only available with products on the CaseWareCloud SE platform.

Learn more about Smart Engagement products.

When you're assigned an issue from an online engagement, you will typically be assigned to a specific document, checklist, line or engagement. The benefit of creating issues is that you'll always have context within the appropriate engagement file.

Once an issue is addressed, mark it as Resolved. Later, you can clear the issue once it's ready to be closed.

Resolve review notes

After completing your review, you can resolve the issue. You can resolve the issue from the Issue menu or from the associated document.

To resolve an issue:

  1. Select the Issue icon ().

  2. On the Issue pop-up, select Resolve.

Clear review notes

After resolving a review note, or if the review note is no longer required, you can clear it from the issues menu. Clearing the review note will remove the notification on the issues icon, while maintaining the history of the note in the engagement file.

To clear a review note:

  1. Select the Issue icon ().

  2. On the Issue pop-up, select Clear.

Complete review notes in an engagement file (Review on the Go)

This feature is only available with CaseWare Working Papers.

Learn more about CaseWare Working Papers.

In Review on the Go, review notes appear as Issues. If you’ve been assigned an issue in a Review on the Go document, you’ll need to resolve it from within the synced Review on the Go snapshot.

To clear review notes in Review on the Go:

  1. Ensure that you have the Entity Access role or equivalent rights.

  2. Select the Review on the Go document you’ve been assigned to.

  3. Select the Issues tab to bring up all issues assigned to you for the engagement.

  4. Select an issue’s name to see the description. To clear the issue, update the issue’s Status to Completed. You may want to add a Resolution.

  5. Select Save to clear the issue.
