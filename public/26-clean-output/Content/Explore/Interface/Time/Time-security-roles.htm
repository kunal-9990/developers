Time role requirements

Time has an integrated security model with CaseWare Cloud. The existing object types in Cloud will extend to features in Time.

The following table outlines the role requirements to perform actions in Time.

Area Action Minimum built-in role Permissions
Time and Expense Entering own Time and Expense - Own No permissions required
Entering on client Entity Access role on entity Read permission on Entity - Summary object
Entering for other staff Time and Expense - All role Read permission on Staff - Details object
Editing pending entries for other staff Time and Expense - All role Edit permission on Time and Expense - All Staff object
Analysis Viewing own - No permissions required
Viewing other staff Staff Admin role Read permission on Staff - Details object
Filtering by staff - No permissions required.
Filtering by client Entity Access role on entity Read permission on Entity - Summary object
Staff rates Viewing staff rates Staff Admin role Read permission on Staff - Details object
Editing staff rates Staff Admin role Edit permission on Staff - Details object
Work codes Viewing or Editing Work Codes Settings Admin role Edit permission on Organization Profile object
