CaseWare Q FAQ

This topic discusses some of the frequently asked questions (FAQs) and known issues when using CaseWare Q.

  • How should my organization use Q?

    An administrator at your organization should set up the Q firm settings in the organization's Q integrated template. When creating a new engagement file from the template, or during the planning stage of the Audit, a senior person should populate the engagement information on the Q file. By completing both parts of Q, and publishing the client file to Cloud, the Q detail will auto-populate as the audit progresses.

  • Will a user have to open up Q at the end of each audit phase and update it?

    No. Once the initial setup mentioned above is complete, no further input is required.

  • Where can I get more information on when to open Q and when to complete each area?

    Once the initial setup is complete, all areas in Q will populate automatically. If content is not updating, please contact your administrator for assistance.

  • Can I set up multiple Q documents for different types of engagements and statuses?

    Yes, you can add multiple Q documents to the master template. When creating a new engagement file from the template, select which one applies and remove the rest.

  • What is the purpose of the second row of annotations in Settings | Status?

    The second row of annotations corresponds to the sign-off event for the role selected in the second drop-down.

    Use when requiring a second sign-off event for a document already annotated in the first row, or for a different document related to the same status that requires a unique role sign-off.
