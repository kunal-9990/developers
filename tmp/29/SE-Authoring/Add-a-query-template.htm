Add a query template

You can use Query documents to collect information from clients. You can create a standard for client requests by creating Query templates. Users can add a Query document in the engagement file using the query template.

Best Practice: We recommend that you create query documents from the template for users rather than having them do it themselves. This way, you can include the query document in the workflow of the engagement file template and apply visibility logic for it.

A query template (or document) consists of the following components:

  • Queries - The questions that the clients need to respond to.

  • Groups- To group related queries.

  • Response Types - To specify the expected user response. The following table describes the available response types:

    Response type

    Expected user response

    Manual

    Enter the requested information.

    Picklist

    Select one item from the drop-down list.

    Multi-Picklist

    Select one or more items from the drop-down list.

    Upload Files

    Browse and add a requested file (or drag and drop the file).

    Date

    Select a date from the calendar.

  • Guidance - To provide guidance information for the users. For example, add the supported file types for a File Upload request.

To add a query template:

  1. Select Add () at the documents page, then choose Query Template.

    2. A blank document opens with a guidance section at the top.

  2. Select Edit () to start authoring.

To add a query document:

  1. Select Add () at the documents page, then choose Query.

  2. Select an existing query template to create your query document from.

  3. Add a name for the query document.

    4. You can optionally add a Number for the document.

  4. Select Create Query.

    5. A blank document opens with a guidance section at the top.

  5. Select Edit () to start authoring.
