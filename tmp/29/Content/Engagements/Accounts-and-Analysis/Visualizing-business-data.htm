Getting an overview of your client data

In your engagement’s Profile page, Analytics displays summary values based on the data contained within the uploaded client files. This summary includes a journal, ledger, and master file data, but not subledgers (such as accounts payable and accounts receivable). Select Profile at the top left of Analytics to access it.

This overview includes:

  • The date range for each journal and ledger file.
  • The number of users in each journal and ledger file.
  • The total number of transactions and transaction Items in each journal and ledger file.
  • The total, largest, smallest, and average amounts for each journal and ledger file.
  • The number of cities and suppliers or customers in each master file.

Data files in .zip format display these summary values automatically. If your engagement contains .csv format data files that have not been mapped, Analytics won’t be able to “read” your files for these summary values until you map certain columns.

To display an overview of a .csv format data file:

  1. Ensure that you have the Editor role or equivalent privileges.

  2. From the Cloud menu, select Analytics.

  3. Select the appropriate Analytics engagement.

  4. On the bottom of the engagement’s Profile page, select more information is required next to an unmapped uploaded file to open the file’s Profile Mapping dialog

  5. Select a CSV File Type from the drop-down menu.

  6. Select each column’s Not Yet Mapped drop-down to select the appropriate column meaning. If none apply, leave it unmapped.

  7. When all possible columns have been mapped, select Save.

Your mapped file will appear on the Profile page. Its position is determined by the Type you selected.

Viewing and downloading charts

At the far right of the Profile area, you’ll see a set of chart icons (). These can be selected to access a set of overview charts for all journal and ledger files.

The available charts are:

  • Transactions Value vs Count Overview by Month - total transaction value and count per month
  • Transactions Value vs Count Overview by User - total transaction value and count per user
  • Transactions by Transaction Date - all transactions by date (x-axis) and amount (y-axis), represented as a scatter plot
  • Number of Transactions by Transaction Amount - groups transactions by amount range and provides a count for each range

The individual values in the Overview charts can be selected to drill down on the data and view detailed information about the data in table form.

To download any of these charts, select the chart's menu and select the appropriate option.

Visualizing test results

Some Analytics tests provide results in chart form. These charts can be saved to your Analytics engagement, then downloaded to your computer or directly to a Working Papers engagement.

  • Compare users
  • Out-of-balance entries
  • Gaps between date columns
  • Accounts receivable aging by date
  • Accounts payable aging by invoice
  • Customers/Debtors with net credit balances
  • Entries exceeding a specified threshold (.csv data files only)
  • Benford’s law (.csv data files only)

To view a test result in graph form:

  1. Ensure that you have the Editor role on or equivalent privileges.

  2. From the Cloud menu, select Analytics.

  3. Select the appropriate data file from the Data Files pane on the left.

  4. Select the Tests tab. A set of test categories will be displayed. Tests with a graph icon () can produce a graph result.

  5. Select the arrow next to a test to run it. A green arrow () indicates that you can run the test without providing any parameters or additional mapping. If a test displays a yellow () or dark gray () arrow, you need to select Edit and provide additional parameters or map additional columns before you can run the test.

  6. On the test results page, select the Graphs tab to display the graph version of the test result.

  7. To save the test result to your Analytics engagement, select Save. The saved test result will appear in the Data Files pane as a child of the original data file.

 

The chart of your test result is saved to Analytics.

Downloading a chart from a test result

Once you've generated your test result, you can download it to your computer, or directly to a Working Papers engagement.

To download an Analytics report to your computer:

  1. Ensure that you have the Editor role on the file or equivalent privileges.

  2. From the Cloud menu, select Analytics and then select the appropriate Analytics engagement.

  3. Next to your saved report, select More Actions () | Download and select the appropriate download option. When a graph is available, you can download it in .png format.

The test result is downloaded to your computer.

To download an Analytics report to a Working Papers engagement:

  1. Ensure you have the Editor role on the file or equivalent privileges.

  2. In Working Papers, click File | Options | CaseWare Cloud. Sign in with your Cloud credentials.

  3. Select Cloud | Engagement, Entity, or Files to open Cloud in your Working Papers browser window.

  4. In the browser window, select Analytics from the Cloud menu.

  5. Next to your saved report, select More Actions() | Download and select the appropriate download option. When a graph is available, you can download it in .png format.

  6. In the File Download dialog, click Add.

  7. In the New Document Link dialog, specify the appropriate settings for the document and click OK.

You have downloaded the report to Working Papers. The report file appears in your Working Papers Document Manager.
